# Task 03：他测模式 — 关系选择 + 观察型题库 + 算分

## 前置
- 已完成 task01、task02
- 请先读取 `CLAUDE.md` 和 `docs/codebase-map.md`

## 目标
实现「我眼中的 ta」的完整答题流程：选关系 → 30 题观察型题目 → 复用原版算分引擎得出人格结果。

## 步骤

### 1. 关系选择界面

在 `other-test.html` 中实现：

**页面开头：**
- 标题：「你眼中的 ta 是什么人」
- 提示语：「选一个关系，替 ta 答 30 道题」
- 免责：「这是你视角下的 ta，不代表 ta 本人，仅供娱乐」

**关系按钮（7 个）：**

| 关系 | 显示文字 | 内部值 |
|------|---------|--------|
| 朋友 | 我眼中的朋友 | friend |
| 暧昧对象 | 我眼中的暧昧对象 | crush |
| 前任 | 我眼中的前任 | ex |
| 现任 | 我眼中的现任 | partner |
| 同事 | 我眼中的同事 | colleague |
| 老板 | 我眼中的老板 | boss |
| 室友 | 我眼中的室友 | roommate |

**选择关系后：**
弹出一个输入框「给 ta 起个代号（可选）」，默认值按关系自动填（如"我朋友"、"某前任"），用户可修改。点击「开始」进入答题。

将 `关系类型` 和 `代号` 存入 localStorage。

### 2. 创建观察型题库

新建 `js/questions-other.js`（或 `data/questions-other.js`，保持与原版题库文件同级）。

题库包含 30 道观察型题目，覆盖 15 个维度（每维度 2 题）。

**数据结构与原版自测题库保持完全一致**，只替换题目文本和选项文本。分值映射与原版相同（每题 4 个选项，分值 1~3，具体分值参照原版结构）。

**第一批 15 题（每维度第 1 题）** 直接使用以下内容：

#### S1 自尊自信
```
题目：ta 被当面否定后，更常见的反应是：
A. 当场反驳，据理力争（3分）
B. 表面不在意，回头琢磨半天（2分）
C. 笑笑就过了，懒得争（2分）
D. 明显受伤，但嘴上不说（1分）
```

#### S2 自我清晰度
```
题目：ta 做重大决定（换工作、分手等）时的风格是：
A. 突然宣布已经决定了，谁也拦不住（3分）
B. 跟很多人聊，但最后还是自己拿主意（2分）
C. 反复纠结，今天想这样明天想那样（1分）
D. 别人怎么说就怎么做（1分）
```

#### S3 核心价值
```
题目：ta 花钱的风格更像：
A. 对自己很舍得，享受型消费不眨眼（3分）
B. 该花花该省省，有自己的优先级（2分）
C. 对别人大方，对自己抠门（2分）
D. 什么都舍不得，除非迫不得已（1分）
```

#### E1 依恋安全感
```
题目：ta 超过半天没回消息时，你觉得更可能是：
A. 真有事，忙完就回（3分）
B. 看到了但晚点回（2分）
C. 不想回（1分）
D. 心情不好在躲人（1分）
```

#### E2 情感投入度
```
题目：ta 在关系里更常被人吐槽的是：
A. 太热情了，有时让人有压力（3分）
B. 挺上心的，但不太会表达（2分）
C. 若即若离，忽冷忽热（1分）
D. 好像谁都无所谓（1分）
```

#### E3 边界与依赖
```
题目：ta 跟最亲近的人闹矛盾后，更常见的状态是：
A. 当场摊开，不过夜（3分）
B. 冷处理一阵子再主动找（2分）
C. 嘴上说没事，但明显在闹别扭（1分）
D. 直接消失一段时间（1分）
```

#### A1 世界观倾向
```
题目：ta 聊到社会新闻或热点事件时，更像哪种风格：
A. 觉得世界还是好人多，总能看到积极面（3分）
B. 理性分析，不太带个人情绪（2分）
C. 动不动就「这个世界就是这样」（1分）
D. 基本不关心，刷到就划走（2分）
```

#### A2 规则与灵活度
```
题目：ta 排队时前面有人插队，更可能的反应是：
A. 直接说「请排队」（3分）
B. 不说但心里骂半天（2分）
C. 看情况，闹太过分才出声（2分）
D. 无所谓，又不差这几分钟（1分）
```

#### A3 人生意义感
```
题目：ta 提到未来或人生目标时，你的感觉是：
A. ta 很清楚自己要什么，而且在行动（3分）
B. 有模糊的方向，但经常变（2分）
C. 嘴上说无所谓，但偶尔能看出 ta 在想（2分）
D. 好像真的不太想这种事（1分）
```

#### Ac1 动机导向
```
题目：如果要用一个词形容 ta 做事的驱动力：
A. 要赢 / 要证明自己（3分）
B. 好奇心 / 觉得好玩（2分）
C. 责任感 / 不得不做（2分）
D. 没什么特别动力，随缘（1分）
```

#### Ac2 决策风格
```
题目：ta 去餐厅点菜的风格是：
A. 打开菜单三秒就决定了（3分）
B. 看一遍菜单，选两三个比一下（2分）
C. 问同行的人「你们点什么」然后跟着选（1分）
D. 翻来覆去犹豫不决，菜上了还在后悔（1分）
```

#### Ac3 执行模式
```
题目：ta 说「明天开始健身 / 学 XX / 早睡」，你的第一反应是：
A. ta 说到做到的概率很高（3分）
B. 大概会做几天然后不了了之（2分）
C. 只是说说，不用当真（1分）
D. ta 这句话已经说过很多次了（1分）
```

#### So1 社交主动性
```
题目：ta 在多人聚会中更像：
A. 全场焦点，到处招呼人（3分）
B. 跟认识的人聊得挺开，不主动认识新人（2分）
C. 安静待着，被搭话才聊（1分）
D. 能不去就不去（1分）
```

#### So2 人际边界感
```
题目：ta 被别人打探隐私时：
A. 礼貌但坚定地拒绝回答（3分）
B. 模糊带过，不正面回应（2分）
C. 虽然不太想说但还是说了（1分）
D. 完全不在意，什么都可以聊（2分）
```

#### So3 表达与真实度
```
题目：你觉得 ta 在社交场合展现的样子和私下差别大吗：
A. 几乎没差别，ta 什么场合都是那个样（3分）
B. 有一些调整，但大体一致（2分）
C. 差挺多的，像两个人（1分）
D. 我也不确定哪个是真的 ta（1分）
```

**第二批 15 题（每维度第 2 题）** 需要你自行撰写，要求：
- 同样是观察型 / 行为型题目（基于外显行为，不问内心感受）
- 维度映射和分值结构与第一题保持一致
- 场景尽量不重复，语气保持荒诞轻松
- 每题 4 个选项，分值分配参照原版结构

### 3. 答题交互

在 `other-test.html` 中实现答题流程：
- 选完关系后进入答题
- UI 交互方式与原版自测保持一致（单选、进度条、可回退）
- 30 题全部使用 `questions-other.js` 中的题库
- 题目上方显示「你眼中的 [代号]」提醒用户在替谁答题

### 4. 算分 — 复用原版引擎

**关键：不要重写算分逻辑，直接复用。**

- 找到原版的算分函数（在 task01 的 codebase-map 中已定位）
- 将其提取为独立可调用的模块（如果尚未独立的话）
- 他测答题数据走同一套管道：30 题 → 15 维度求和 → L/M/H 映射 → 曼哈顿距离匹配
- 将他测的 15 维向量和匹配结果存入 localStorage：
  ```javascript
  localStorage.setItem('dbti_other_result', JSON.stringify({
    relation: 'ex',          // 关系类型
    nickname: '某前任',       // 代号
    vector: [2,3,1,...],     // 15维向量（数值化后的）
    levels: 'HML...',        // 15维的 H/M/L 串
    type: 'FAKE',            // 匹配的人格类型
    timestamp: Date.now()
  }));
  ```

### 5. 答完跳转

答完 30 题、算分完成后，跳转到 `result-other.html`（task04 实现结果展示，本 task 先做一个简单的临时结果页，显示匹配到的人格类型名称即可）。
