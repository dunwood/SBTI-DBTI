# Task 02：改造首页为双入口 + 自测流程独立化

## 前置
- 已完成 task01，`docs/codebase-map.md` 中有完整的代码结构分析
- 请先读取 `CLAUDE.md` 和 `docs/codebase-map.md`

## 目标
把首页改造成「我测我自己」+「我眼中的 ta」双入口，同时保证自测流程完全不受影响。

## 铁律
**改造过程中自测功能必须全程可用。如果原版是单页应用，需要小心拆分，不要破坏自测链路。**

## 步骤

### 1. 规划页面拆分方案

根据 task01 的代码分析，确定改造方案。核心原则：
- 原版自测流程的代码逻辑**不动**，只调整入口路由
- 新建他测相关页面，复用算分引擎

预期页面结构：
```
index.html          → 新首页（双入口）
self-test.html      → 自测模式（原版流程迁移至此，或保持原位不动）
other-test.html     → 他测模式（task03 实现，本 task 先建空壳）
result-self.html    → 自测结果（可能就是原版结果页，改名或保持）
result-other.html   → 他测结果（task04 实现，本 task 先建空壳）
```

**注意：** 如果原版是单个 index.html 里的单页应用（通过 JS 控制显示/隐藏不同区块），那么有两种策略：
- **策略 A（推荐）：** 保留原版 index.html 不动，改名为 self-test.html；新建 index.html 作为双入口首页，链接到 self-test.html
- **策略 B：** 在原版 index.html 中增加一层入口选择界面，点击「我测我自己」后进入原有流程

请根据实际代码结构选择最安全的策略，**优先选不动原有代码的方案**。

### 2. 实现新首页 index.html

设计要求：
- 暗色背景，风格与原版一致
- 页面居中，极简布局
- 标题：`DBTI 人格镜像测试`
- 副标题：`你以为的 ta，和 ta 以为的自己`
- 两个大按钮并排（或上下排列适配手机）：
  - 「我测我自己」→ 链接到自测页面
  - 「我眼中的 ta」→ 链接到 other-test.html（暂时可以显示"即将上线"）
- 免责提示：`仅供娱乐，好玩为主，不要当判决书`
- 底部署名区域：

```
原创：蛆肉儿串儿（B站 UID417038183）
开源逆向：pingfanfan/SBTI
关系模式 & 他测玩法：哲学园 · 拾柒年蝉 出品
```

### 3. 建立他测空壳页面

创建以下文件，内容为简单的占位页面（暗色背景 + 标题 + "开发中，敬请期待" + 返回首页链接）：

- `result-other.html`
